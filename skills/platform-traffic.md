# 平台流量运营

> 公域流量全流程：选题 → 创作 → 发布 → 数据回收。覆盖公众号、小红书、抖音、视频号等平台。

---

## 触发条件

- 用户说"帮我出选题"、"生成选题"、"本周选题"
- 用户说"写文章"、"写内容"、"做一篇"
- 用户提到具体平台（公众号/小红书/抖音/视频号/Blog）
- 用户说"这篇发了"、"已发布"、"数据怎么样"

---

## 一、工作流程

### Step 1：了解用户（仅首次）

首次接到任务时，先读取以下文件了解用户的品牌和业务：
- `{VAULT_ROOT}/00-me/10-定位与品牌/IP定位.md` — 用户是谁、做什么
- `{VAULT_ROOT}/00-me/10-定位与品牌/品牌风格指南.md` — 语气风格、口头禅、禁用词
- `{VAULT_ROOT}/00-me/20-产品与业务/产品与服务清单.md` — 卖什么
- `{VAULT_ROOT}/00-me/30-目标客户/目标客户画像.md` — 写给谁看
- `{VAULT_ROOT}/00-me/50-目标与节奏/日常工作节奏.md` — 发布频率和时间

### Step 2：选题生成

#### 2.1 扫描素材库
查看以下目录获取选题灵感：
- `{VAULT_ROOT}/10-raw/` — 最近收集的素材
- `{VAULT_ROOT}/20-wiki/30-客户洞察/` — 客户痛点 → 内容选题
- `{VAULT_ROOT}/20-wiki/50-内容与流量/爆款内容拆解.md` — 爆款规律参考
- `{VAULT_ROOT}/20-wiki/60-实战复盘/成功案例库.md` — 自己的成功经验

#### 2.2 选题四类比例
- **60% 硬核干货** — 方法论、工具教程、实操流程
- **30% 认知趋势** — 行业观察、观点输出、趋势分析
- **10% 真实日常** — 工作日常、踩坑复盘、个人故事

#### 2.3 创建选题卡
路径：`{VAULT_ROOT}/30-flow/10-公域流量/10-选题池/`
命名：`📋选题中-{YYYY-MM-DD}-{选题标题}.md`

```yaml
---
title: "{选题标题}"
status: 选题中
type: 选题卡
platform: {目标平台}
created: {YYYY-MM-DD}
tags: [标签]
source: ""
---
```

```markdown
# {选题标题}

## 核心观点
<!-- 一句话说清这篇内容要表达什么 -->

## 推荐形式与平台
- [ ] 文章（1200-2000字）→ 公众号
- [ ] 图文（500字+配图）→ 小红书
- [ ] 口播稿（200-300字）→ 抖音 + 视频号
- [ ] 短帖（200-400字）→ 朋友圈

## 大纲
1.
2.
3.

## 目标读者
<!-- 谁最需要看这篇内容 -->
```

### Step 3：内容生产

#### 3.1 选题决定形式
- **深度分析 / 方法论** → 文章（1200-2000字）→ 公众号
- **干货步骤 / 工具对比** → 图文（500字+配图）→ 小红书
- **观点输出 / 热点点评** → 口播稿（200-300字）→ 抖音 + 视频号
- **碎片思考 / 互动** → 短帖（50-200字）→ 朋友圈

#### 3.2 创作前查阅知识库
根据选题方向，按需调用：
- `{VAULT_ROOT}/20-wiki/30-客户洞察/` — 客户痛点
- `{VAULT_ROOT}/20-wiki/40-销售与转化/` — 谈单方法论和话术
- `{VAULT_ROOT}/20-wiki/50-内容与流量/` — 内容运营方法论
- `{VAULT_ROOT}/20-wiki/10-行业与市场/` — 行业趋势
- `{VAULT_ROOT}/20-wiki/20-产品知识/` — 产品专业知识

#### 3.3 创建内容文件
路径：`{VAULT_ROOT}/30-flow/10-公域流量/20-内容管理/{平台}/`
命名：`📝草稿-{YYYY-MM-DD}-{标题}.md`

### Step 4：发布状态管理

#### 4.1 状态流转
```
📋选题中 → 📝草稿 → 📤待发布 → ✅已发布 → 🚫Closed
```
视频额外：`📝草稿 → 🎥拍摄中 → ✂️剪辑中 → 📤待发布`

#### 4.2 发布操作
当用户说"这篇发了"、"已发布"时：
1. 将文件名 emoji 改为 ✅
2. 更新 frontmatter：`status: 已发布`，`updated: {今天日期}`
3. 不移动文件位置

#### 4.3 各平台内容目录
| 平台 | 路径 |
|------|------|
| 公众号 | `30-flow/10-公域流量/20-内容管理/公众号/` |
| 小红书 | `30-flow/10-公域流量/20-内容管理/小红书/` |
| 抖音 | `30-flow/10-公域流量/20-内容管理/抖音/` |

## 二、内容规范

### 语气要求
- 严格遵循品牌风格指南中的语气基调
- 使用用户习惯的口头禅和表达方式
- 绝不使用禁用词

### 结构要求
- 钩子开头 → 要点展开 → 案例/数据 → 结尾引导行动
- 内容要有信息增量，不给读者制造焦虑
- 提供可执行的方案

### 原创性
- 不抄袭，用用户自己的话和案例来表达
- 调用 wiki 中的知识作为素材，但要用自己的语言重写

---

## 三、数据回收

### 触发条件
用户报数据时（如"公众号那篇阅读量500"）。

### 操作
更新已发布文件的 frontmatter 中的 data_7d 字段：
```yaml
data_7d:
  日期: {YYYY-MM-DD}
  # 按平台不同字段不同
  # 公众号：阅读/点赞/在看/转发
  # 小红书：浏览/点赞/收藏/评论
  # 抖音：播放/点赞/评论/转发
  performance: ""  # 自动判定：爆/S/A/B/C
```

### 表现等级判定
- **爆**：远超平时水平（如平时阅读200，这篇2000+）
- **S**：超过平时2倍以上
- **A**：超过平时1.5倍
- **B**：接近平时水平
- **C**：低于平时一半

---

## 四、Pitfalls

- 不要跳过选题直接写内容（除非用户明确指定选题）
- 不要在内容文件中放原始资料（去 wiki 找，用自己话重写）
- 不要连发同类型内容（注意四类比例）
- 状态变更只改文件名和 frontmatter，不要移动文件
- 选题卡和内容文件是两个独立文件，不要合并
