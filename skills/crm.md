# 客户关系管理（CRM）

> 客户全生命周期管理：从线索录入到成交归档，保持沟通记录完整，不丢单。

---

## 触发条件

- 用户口述新客户信息（"加了个人"、"新线索"）
- 用户口述跟进情况（"今天跟XX聊了"、"跟进一下"）
- 用户报成交（"签了"、"成交了"、"打款了"）
- 用户查询客户（"XX客户怎么样了"）

---

## 一、工作流程

### Step 1：了解业务（仅首次）

读取以下文件：
- `{VAULT_ROOT}/00-me/20-产品与业务/产品与服务清单.md` — 卖什么、价格多少
- `{VAULT_ROOT}/00-me/20-产品与业务/价格体系.md` — 价格策略
- `{VAULT_ROOT}/00-me/30-目标客户/目标客户画像.md` — 目标客户特征
- `{VAULT_ROOT}/00-me/30-目标客户/客户分层标准.md` — 分层规则
- `{VAULT_ROOT}/20-wiki/40-销售与转化/` 目录下所有文件 — 谈单方法论和话术

### Step 2：创建线索卡

#### 触发

#### 路径

```yaml
  name: ""
  company: ""
  wechat: ""
  phone: ""
  备注: ""
---
```

```markdown
# {客户姓名/公司简称}

## 客户基本信息
<!-- 根据用户提供的信息填写 -->

## 需求分析

<!-- 根据分层标准评估 A/B/C/D -->

<!-- 关键信息、偏好、决策因素等 -->
```

### Step 3：记录跟进

#### 触发
用户口述与客户的沟通情况。

#### 操作
1. 找到对应的线索卡
2. 在「跟进记录」部分追加新记录（最新的放在最上面）
3. 根据沟通情况判断是否需要更新客户分层
4. 更新 status 字段
5. 更新 `updated` 日期为当天
6. 给出下一步跟进建议（基于销售方法论和话术库）

#### status 流转
```
新线索 → 跟进中 → 意向确认 → 报价中 → 谈判中 → 已成交
                                                    ↓
                                                已流失 → Closed
```

### Step 4：成交归档

#### 触发
用户报成交。

#### 操作
1. 更新 status 为「已成交」
2. 在线索卡末尾追加成交信息：
```markdown
## 成交信息
- **成交日期**：
- **成交金额**：
- **成交产品/方案**：
- **成交渠道**：

## 成交复盘
<!-- 什么做对了？什么可以改进？ -->
```
3. 将文件移到 `{VAULT_ROOT}/30-flow/30-CRM/20-成交记录/`
4. 文件名改为：`✅已成交-{YYYY-MM-DD}-{客户姓名}.md`

## 二、客户分层标准

| 等级 | 特征 | 跟进频率 |
|------|------|----------|
| **A级** | 明确需求 + 有预算 + 有决策权 | 1-3天/次，重点跟进 |
| **B级** | 有需求 + 有预算，但需培育 | 1周/次，持续价值输出 |
| **C级** | 有潜在需求，未到决策时机 | 2周/次，保持触达 |
| **D级** | 暂不匹配，优先级低 | 1月/次，偶尔触达 |

---

## 三、工作原则

- **每次跟进后立即记录** — 不要等，忘掉 = 丢单
- **跟进记录要具体** — 不是「聊了一下」，而是「沟通了XX问题，客户表示YY」
- **主动给建议** — 基于销售方法论，告诉下一步该怎么推进
- **分层动态调整** — 客户分层不是一次性的，根据沟通情况随时调整
- **成交后也要维护** — 成交不是结束，是复购和转介绍的开始
- **注意隐私** — 所有客户信息仅用于业务管理

---

## 四、跟进建议生成

当用户问"这个客户怎么跟"或"下一步怎么做"时：

1. 读取线索卡的完整跟进历史
2. 读取销售方法论和话术库
3. 分析客户当前状态和卡点
4. 给出 2-3 个具体的下一步行动建议
5. 建议应该包含：说什么、做什么、什么时候做

---

## 五、Pitfalls

- 不要编造客户信息 — 只记录用户口述的内容
- 不要自动改状态为"已成交" — 必须用户明确说成交了
- 不要在成交后删除线索卡 — 移到成交记录目录
- 已成交的客户不需要跟进提醒
- 跟进记录用倒序（最新的在最上面）
- 不要把" Closed won" 的客户再提示跟进
