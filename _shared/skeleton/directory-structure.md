# Vault 目录结构

## 一键创建命令

```bash
mkdir -p "$VAULT"/{00-me/{10-定位与品牌,20-产品与业务,30-目标客户,40-账号矩阵,50-目标与节奏},10-AI/{10-schemas,20-templates,30-dataview,40-automations,50-mcp,60-prompts,70-integrations,80-rules},20-raw/{inbox,cleaned,tagged},30-wiki/{10-行业与市场,20-产品知识,30-客户洞察,40-销售与转化,50-内容与流量,60-实战复盘},40-flow/{10-公域流量/{10-选题池,20-内容管理,30-数据统计,40-内容资产,50-素材提炼},20-私域运营/{10-朋友圈,20-社群,30-数据周报},30-CRM/{10-线索管理,20-成交记录},40-产品交付,45-项目管理,50-人事管理,50-财务管理,55-库存管理,60-数据看板},50-assets/{品牌,账号,内容模板,PPT模板,报价方案,合同},60-events/{10-company,20-ai},99-archive}
```

## 八层结构说明

```
00-me/        # 人：决策层——我是谁、卖什么、卖给谁、目标与节奏
10-AI/        # AI：智能层——schemas、templates、dataview、automations、MCP、prompts、integrations、rules
20-raw/       # 输入层：素材入口（只进不改）
30-wiki/      # 知识层：消化后的认知（只放结论）
40-flow/      # 业务流程层：各部门日常运转（只做调用）
50-assets/    # 资产层：品牌、账号、模板、合同、报价方案
60-events/    # 事件日志：公司大事记 + AI事件记录（按时间线）
99-archive/   # 归档：已结束的内容和项目
```

## 设计哲学

- **00-me + 10-AI**：人与 AI 并列在最顶层，人做决策，AI 做执行
- **20-raw → 30-wiki → 40-flow**：信息流水线——输入 → 消化 → 输出
- **40-flow**：各部门的操作台，包含市场运营、CRM、人事、财务、库存、项目管理
- **50-assets**：公司核心资产，品牌和账号也归入此类
- **60-events**：时间线日志，记录公司里程碑和 AI 异常复盘
- **99-archive**：所有事物的最终归宿
