# Vault 目录结构

## 一键创建命令

```bash
mkdir -p "$VAULT"/{00-me/{10-定位与品牌,20-产品与业务,30-目标客户,40-账号矩阵,50-目标与节奏},10-raw/{inbox,cleaned,tagged},20-wiki/{10-行业与市场,20-产品知识,30-客户洞察,40-销售与转化,50-内容与流量,60-实战复盘},30-flow/{10-公域流量/{10-选题池,20-内容管理,30-数据统计,40-内容资产,50-素材提炼},20-私域运营/{10-朋友圈,20-社群,30-数据周报},30-CRM/{10-线索管理,20-成交记录},40-产品交付,45-项目管理,50-人事管理,50-财务管理,55-库存管理,60-数据看板},40-agents/{skills,schedules,10-内容创作,20-调研分析,30-运营管理,40-效率工具},50-projects/{客户,品牌,账号},60-assets/{内容模板,PPT模板,报价方案,合同},99-archive}
```

## 九层结构说明

```
00-me/        # 决策层：我是谁、卖什么、卖给谁
10-raw/       # 输入层：素材入口（只进不改）
20-wiki/      # 知识层：消化后的认知（只放结论）
30-flow/      # 业务流程层：日常干活（只做调用）
40-agents/    # AI能力层：AI助手配置
50-projects/  # 实体层：客户/品牌/账号资料
60-assets/    # 可复用资产层：模板、合同、报价方案
99-archive/   # 归档：已结束的内容和项目
```
