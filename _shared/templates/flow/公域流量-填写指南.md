# 内容管理 — 按平台存放内容文件

## 这个文件夹是干什么的
所有公域内容的创作和存放区。文件从生到死不搬家，状态变更只改文件名前缀。

## 目录结构
在这个目录下，按你的平台创建子目录，例如：
- `公众号/` — 公众号文章（1200-2000字长文）
- `小红书/` — 小红书图文（500字）
- `抖音/` — 抖音口播稿（200-300字）
- `视频号/` — 视频号口播稿
- `官网blog/` — 博客文章

## 文件命名规则
`{状态emoji}-{日期YYYY-MM-DD}-{标题}.md`

状态流转：
- 图文/文章：📋 → 📝 → 📤 → ✅ → 🚫
- 视频：📋 → 📝 → 🎥 → ✂️ → 📤 → ✅ → 🚫

## Frontmatter 规范
每个内容文件都要有 frontmatter：
```yaml
---
title: 文章标题
created: 2026-05-05
status: 草稿
platform: 公众号
tags: []
data_7d:
  日期: null
  阅读: null
  点赞: null
  在看: null
  转发: null
performance: null
---
```

## 注意事项
- 文件从生到死不搬家，状态变更只改 frontmatter + 文件名前缀
- 发布7天后报一次数据，写入 data_7d，自动判定 performance（爆/中/扑）
- 每个平台的数据字段不同：公众号用"阅读/在看"，小红书用"浏览/收藏"，抖音用"播放"
