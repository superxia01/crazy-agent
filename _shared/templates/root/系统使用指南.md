# 系统使用指南

## 这个Vault怎么用

这是一个帮你管理个人IP + 自媒体运营 + 获客转化的知识库。所有文件分9层，每层只做一件事。

## 九层架构速览

```
00-me/        → 你是谁、卖什么、卖给谁（填这里）
20-raw/       → 看到好东西就往这扔（素材入口）
30-wiki/      → 你消化后的认知和经验（知识沉淀）
40-flow/      → 日常干活的地方（选题/创作/发布/CRM）
10-AI/    → AI工具配置和使用（可选）
40-flow/  → 客户/品牌/账号资料（实体管理）
50-assets/    → 可复用的模板和资产（拿来就用）
99-archive/   → 不用的东西放这里（保持vault清爽）
```

## 三条铁律

### 1. raw = 只进不改
看到好文章、好视频、行业报告、竞品截图 → 全部扔到 `20-raw/inbox/`。
- 不需要分类，不需要整理，不需要打标签
- 定期清理：有价值的移到 `cleaned/` 或 `tagged/`，没用的删掉
- raw 里永远是"别人的东西"，不是你的知识

### 2. wiki = 只放结论
从 raw 里看到有价值的内容，消化之后用自己的话写进 wiki。
- 判断标准：如果你用自己的话重新写不出来，说明还没消化，放回 raw
- wiki 里不放链接、不放截图、不放"收藏"——只放你自己写的东西
- 6个分类按问题分：行业市场、产品知识、客户洞察、销售转化、内容流量、实战复盘

### 3. flow = 只做调用
在 flow 里干活时，去 wiki 找素材和知识，不要在这里写知识。
- 写文章？去 wiki/客户洞察 找痛点，去 wiki/内容与流量 找方法论
- 跟客户？去 wiki/销售与转化 找话术，去 wiki/客户洞察 找痛点
- flow 里的文件是"产出"，不是"知识"

## 每天怎么用

### 日常流程（30分钟）
1. 看到好内容 → 扔进 `20-raw/inbox/`（2分钟）
2. 有灵感 → 扔进 `20-raw/inbox/`（1分钟）
3. 发内容 → 在 `40-flow/10-公域流量/` 写/发（10-20分钟）
4. 跟客户 → 在 `40-flow/30-CRM/` 更新线索（5分钟）

### 每周流程（1-2小时）
1. 清理 raw inbox → 有价值的移到 cleaned 或提炼到 wiki
2. 在 `40-flow/10-公域流量/10-选题池/` 规划下周选题
3. 更新 CRM 跟进计划
4. 填写数据周报 `40-flow/20-私域运营/30-数据周报/`

### 每月流程（半天）
1. 回顾 wiki 里有什么需要更新
2. 复盘内容数据，好的进 `40-flow/10-公域流量/40-内容资产/`
3. 检查商业目标进度 `00-me/50-目标与节奏/商业目标.md`

## 从哪里开始

**第一次用这个Vault，按这个顺序来：**

1. 填 `00-me/10-定位与品牌/IP定位.md` — 你是谁
2. 填 `00-me/20-产品与业务/产品与服务清单.md` — 你卖什么
3. 填 `00-me/30-目标客户/目标客户画像.md` — 卖给谁
4. 填 `00-me/40-账号矩阵/平台账号信息.md` — 你有哪些号
5. 把现有的收藏、笔记全部丢进 `20-raw/inbox/`
6. 开始在 `40-flow/10-公域流量/` 创建内容

**不用一次填完。** 先把1-4填了就能开始干活，wiki 和其他配置可以边用边补。

## 文件命名规范

- 内容文件用 emoji 前缀标记状态：📋选题中 → 📝草稿 → 📸拍摄中 → ✂️剪辑中 → 📤待发布 → ✅已发布 → 🚫Closed
- 模板文件以 `_template-` 开头
- wiki 文件名就是主题名，不加前缀
- 00-me 文件名就是内容名，不加前缀

## 信息流转路径

```
看到好内容 → 20-raw/inbox/（扔进去）
                  ↓ 定期清理
           有价值的 → 消化提炼
                  ↓
           30-wiki/（用自己的话写结论）
                  ↓
           40-flow/（干活时调用wiki知识）
                  ↓
           40-flow/（落地到具体客户/项目）
                  ↓
           50-assets/（好的内容沉淀为可复用资产）
```

## 支持的平台

内容由 variant 决定，搭建时自动配置对应平台列表。

## 这个Vault不用什么工具

不需要任何特定的AI工具、插件或付费服务。一个Obsidian就够了。
如果你用其他笔记软件（Notion/飞书文档/语雀），同样可以用这套结构。
