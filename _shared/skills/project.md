## 2. 项目状态流转

```
📐 立案 → 🔵 规划中 → 🟢 进行中 → 🟡 测试中 → 🟣 待验收 → ✅ 已完成
                           ↘               ↘
                          ⏸️ 已暂停          ❌ 已取消
```

## 3. 工作流程

### 3.1 项目立项

1. 收集项目基本信息（名称、客户、目标、预算）
2. 分配项目编号：`PRJ-XXX`
3. 创建项目卡片文件
4. 在 `项目索引.md` 中注册
5. 创建任务列表模板

### 3.2 任务分解

1. 从项目目标拆解为可执行任务
2. 每个任务分配：负责人、优先级、预估工时、截止日期
3. 建立任务依赖关系
4. 确定里程碑节点

### 3.3 进度更新

当用户说"更新项目进度"或"任务完成了"时：

1. 更新对应任务状态
2. 计算项目整体完成率
3. 更新项目卡片中的进度条
4. 检查是否触发里程碑

### 3.4 项目结项

1. 确认所有任务已完成
2. 整理交付物清单
3. 客户验收确认
4. 项目归档到"已完成"目录
5. 复盘总结

## 5. 优先级定义

| 级别 | 标记 | 含义 | 响应时间 |
|------|------|------|----------|
| P0 | 🔴紧急 | 阻塞项目进展 | 立即处理 |
| P1 | 🔴高 | 关键路径任务 | 24小时内 |
| P2 | 🟡中 | 重要但非阻塞 | 3天内 |
| P3 | 🟢低 | 可延后处理 | 本迭代内 |

---

## 6. 注意事项

1. **项目编号**：按创建顺序递增，格式 `PRJ-XXX`
2. **目录命名**：项目目录格式 `PRJ-XXX-项目简称`
3. **进度更新频率**：至少每周更新一次
4. **里程碑变更**：任何里程碑日期变更需记录原因
5. **归档规范**：已完成项目移入"已完成"目录，保留原始结构
6. **工时记录**：工时按人天或小时记录，与财务数据关联
