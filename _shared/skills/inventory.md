## 2. 工作流程

### 2.1 SKU 注册

1. 收集产品信息（名称、规格、成本价、售价）
2. 分配SKU编号：`SKU-XXXX`
3. 创建SKU卡片文件
4. 更新 `SKU总览.md`
5. 设置初始库存数量和安全库存线

### 2.2 订单处理

当用户说"新订单"或"收到订单"时：

1. 记录订单信息（客户、商品、数量、金额）
2. 检查库存是否充足
3. 创建订单文件到"待处理"目录
4. 库存充足 → 进入发货流程；库存不足 → 触发补货预警

### 2.3 发货流程

1. 确认发货信息（收件人、地址、物流方式）
2. 扣减库存数量
3. 更新SKU卡片中的库存数
4. 记录物流单号
5. 订单文件移入"已发货"

### 2.4 库存预警检查

当用户说"检查库存"或"库存预警"时：

1. 遍历所有SKU，对比当前库存与安全库存线
2. 库存 ≤ 安全库存线的SKU列入预警
3. 生成预警报告
4. 建议补货数量（至目标库存量）


## 5. 注意事项

1. **SKU编号**：按创建顺序递增，格式 `SKU-XXXX`
2. **库存实时性**：每次出/入库必须立即更新库存数
3. **安全库存**：根据历史销量设定，建议为周均销量的2-3倍
4. **订单编号**：格式 `ORD-YYYYMMDD-XXX`，按日期+序号
5. **预警频率**：每日检查一次库存预警
6. **供应商管理**：记录主要供应商联系方式和合作条款
