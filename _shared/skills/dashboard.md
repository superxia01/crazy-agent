## 2. 数据来源

本看板自动汇总以下维度数据（需配合其他 Skill 使用）：

| 维度 | 数据来源 Skill | 关键指标 |
|------|---------------|----------|
| 财务 | finance | 收入、支出、利润、现金流 |
| 人事 | hr | 人数、离职率、绩效分布 |
| 项目 | project | 在建项目数、完成率、延期率 |
| 内容 | content | 发布量、阅读量、粉丝增长 |
| CRM | crm | 新客户数、成交金额、转化率 |
| 库存 | inventory | SKU数、库存金额、预警数 |

## 3. 工作流程

### 3.1 生成周报

当用户说"生成本周周报"时：

1. 确定报告周期（本周一 ~ 本周日）
2. 扫描各维度当周数据：
   - 财务：本周收支汇总
   - 项目：本周任务完成情况
   - 内容：本周发布内容统计
   - CRM：本周客户跟进情况
3. 填入周报模板
4. 计算周环比变化
5. 写入 `周报/2026/WXX-周报.md`

### 3.2 生成月报

当用户说"生成本月月报"时：

1. 确定报告周期（本月1日 ~ 月末）
2. 汇总本月各维度数据
3. 与上月数据对比（环比）
4. 与去年同期对比（同比，如有）
5. 分析趋势和异常
6. 提出改进建议
7. 写入 `月报/2026/YYYY-MM-月报.md`

### 3.3 季度复盘

当用户说"季度复盘"时：

1. 回顾季度目标和KPI完成情况
2. 汇总3个月的关键数据
3. 识别亮点和不足
4. 制定下季度计划
5. 写入 `季度复盘/` 目录

### 3.4 KPI 追踪

1. 从年度目标拆解月度KPI
2. 每月更新KPI完成进度
3. 偏差超过20%时标记预警
4. 季度复盘时全面回顾

## 5. 注意事项

1. **数据准确性**：报告数据来源于各模块文件，确保源数据已更新
2. **报告时效**：周报在每周一上午生成，月报在每月1-3号生成
3. **对比维度**：至少提供环比数据，有历史数据时提供同比
4. **达成率标记**：≥100% 🟢达标 / 80-99% 🟡接近 / <80% 🔴未达
5. **行动导向**：报告不仅是数据汇总，必须有改进建议和下期计划
6. **图表建议**：数据量大时建议生成可视化图表辅助说明
